# 7/15 실증 운영 가이드 — 하루 집중, 팀원 전원용

> 목표: **하루 만에 보고서에 쓸 수 있는 데이터를 최대한 확보한다.**
> 참여자 목표: 팀원 1인당 5명 이상, 팀 전체 20명 이상 / 발화 200건 이상.
> 서버 로그가 자동으로 쌓이므로, 종이에는 "로그로 알 수 없는 것"만 적는다.

## 0. 당일 아침 점검 체크리스트 (서버 담당, 5분)

```bash
git pull                          # 최신 코드
bash scripts/serve-tunnel.sh      # 주소 발급 → 단톡방 공유
```

- [ ] `curl localhost:8000/healthz` → `"stt":"clova+gemini"` 인지 확인.
      `"clova"`만 나오면 Gemini 무료 한도가 아직 소진 상태 — CLOVA 단독으로도 실증 가능하니 그대로 진행
      (한도는 매일 한국시간 오후 4~5시경 리셋. 급하면 새 Google 계정에서 키 재발급 → .env의 GEMINI_API_KEY 교체)
- [ ] 태블릿 1대로 리허설 주문 1건 → `data/logs/utterances.jsonl` 마지막 줄에 `"site":` 값 찍히는지 확인
- [ ] 맥 전원 연결 + `caffeinate -dims` 실행 (잠자기 방지)

## 1. 전날 밤 준비 (서버 담당 1명)

```bash
# 맥에서 — 프론트 빌드 + 백엔드 + 공개 HTTPS 주소 발급 (계정 불필요)
git pull
bash scripts/serve-tunnel.sh
# → https://<무작위>.trycloudflare.com 주소가 나온다
```

- **맥은 전원 연결 + 잠자기 금지**: 터미널에서 `caffeinate -dims` 를 하나 더 띄워 둔다.
- 주소는 서버를 다시 켤 때마다 바뀐다 → **실증 당일 아침에 발급해서 단톡방 공유**가 안전.
- ⚠️ 이 주소는 비밀번호가 없다. **단톡방 밖에 공유 금지**, 실증 끝나면 Ctrl+C로 종료.

## 2. 팀원별 태블릿 설정 (실증 시작 전 5분)

1. 태블릿 브라우저(아이패드=Safari, 갤럭시탭=Chrome)에서 접속:
   **`https://<발급주소>/?site=자기이름`** ← site 태그를 꼭 붙일 것 (예: `?site=경로당A_지연`)
   - 이 태그로 서버 로그가 장소·담당자별로 자동 구분된다. 한 번 붙이면 기기에 저장됨.
2. 공유 버튼 → **"홈 화면에 추가"** → 홈 화면 아이콘으로 실행 (전체화면).
3. 마이크 권한 **허용**, 볼륨 최대, 세로 방향 고정.
4. 리허설 1회: "불고기버거 세트 하나 주세요" → 장바구니에 담기면 준비 끝.

## 3. 라즈베리파이 (지연)

```bash
git pull                                  # main 브랜치 — 오늘 수정 전부 포함됨
.venv/bin/pip install -r requirements.txt # pillow 추가됨
cd frontend && npm run build && cd ..
# 이후 docs/raspberry-pi.md 의 서비스 재시작 절차
```

- 파이는 자체 서버라 **로그가 파이 안에 쌓인다** → 실증 끝나면 `data/logs/utterances.jsonl` 회수 필수.
- 화면 회전이 필요하면: `bash scripts/setup-pi-kiosk.sh 90` (부팅마다 자동 적용).

## 4. 참여자 1명 프로토콜 (약 10분, 2인 1조 권장)

| 순서 | 내용 | 기록 |
|---|---|---|
| ① | 인사 + **동의서** (08_실증키트) — 음성이 텍스트로 기록됨을 고지 | 동의서 |
| ② | 과업 1: "드시고 싶은 **버거 세트를 말로** 주문해 보세요" | 성공/개입/포기, 소요시간 |
| ③ | 과업 2: "**음료 하나를 추가**하고 결제까지 눌러 보세요" | 〃 |
| ④ | SUS 설문 5문항 (08_실증키트 축약판) | 설문지 |
| ⑤ | **표현 수집**: "평소 이 메뉴를 뭐라고 부르세요?" 2~3개 | 표현수집표 |

- 진행자는 **재촉·대신 누르기 금지**. 막히면 "천천히 하셔도 됩니다" 한 마디만.
- 실패도 데이터다. 음성이 안 되면 터치로 이어가게 두고 기록지에 "음성→터치 전환"이라고 적는다.

### 종이에만 남는 것 (로그로 알 수 없는 것)
- 과업 성공/실패, 진행자 개입 여부, 포기 지점
- 참여자 반응·표정·혼잣말 ("이건 뭐꼬?" 같은 발화 그대로)
- SUS 점수, 연령대(대략), 키오스크 사용 경험 유무

## 5. 현장 트러블슈팅

| 증상 | 대처 |
|---|---|
| 음성이 계속 인식 안 됨 | 마이크에 20cm 이내로, 주변 소음 줄이기. 2회 실패하면 터치 주문으로 진행(그것도 데이터) |
| "목소리가 잘 들리지 않았어요" 반복 | 태블릿 마이크 구멍이 가려졌는지 확인 (케이스·손) |
| 링크가 안 열림 | 서버 담당에게 연락 → 터널 재발급 → 새 주소 공유 |
| 소리가 안 남 | 화면 아무 데나 한 번 터치 후 다시 시도, 무음 스위치·볼륨 확인 |
| 화면이 대기 화면으로 돌아감 | 45초 무조작 시 정상 동작 — 터치하면 이어짐 |

## 6. 저녁: 데이터 회수 (당일 마감)

```bash
# 1) 파이 로그를 맥으로 복사한 뒤, 서버 로그와 합친다 (그냥 이어붙이면 됨)
cat pi-utterances.jsonl >> data/logs/utterances.jsonl

# 2) 당일 집계 — 장소별(site) 건수까지 자동 집계됨
python scripts/report.py --from 2026-07-15 --to 2026-07-15

# 3) 표현수집표 → data/expressions.yaml 반영 → 전/후 정확도 비교 (보고서 핵심 그래프)
python scripts/measure.py --provider rule          # 교체 후
python scripts/measure.py --provider claude        # 신 메뉴 기준 재측정
```

- 기록지·동의서·설문지는 **당일 밤 사진 촬영**해서 공유 폴더에 업로드 (분실 대비).
