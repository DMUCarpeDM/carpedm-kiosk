# 제품화 브리프 (Design & Product Brief v1)
### "팔아도 손색없는" 수준을 위한 디자인·브랜드·완성도 사양서

> 용도: ① Claude Design 입력용 화면설계 사양 ② 디자인 트랙(김지연·김나우) 작업 기준 ③ 함체·사운드 등 제품 완성도 전 영역의 단일 기준.
> 목표 수준: **양산 직전 시제품** — 실결제·인증·AS는 범위 외(로드맵), 그 외 모든 접점은 상용품처럼.
> ⚠ 표시는 실측·확인 필요. 수치는 팀 설정값(조정 가능).

---

## 1. 제품 정의 & 네이밍

**한 줄**: 기존 키오스크 옆에 놓는, 어르신이 *자기 말로* 주문하는 병설형 음성 주문 창구.

**제품명 후보** (팀 투표로 결정 — 로고·멘트·함체 인쇄에 일관 적용):
| 후보 | 의미 | 강점 | 약점 |
|---|---|---|---|
| **말씀** | 어른의 말을 높이는 우리말. 첫 안내 멘트 "말씀하세요"와 일치 | 공손함이 곧 브랜드. 기억 용이 | 일반명사라 상표 변별력 낮음 |
| **말로** (MALLO) | "말로 하세요" | 짧고 현대적, 로고화 쉬움 | 공손함은 '말씀'보다 약함 |
| **온말** | 따뜻할 온(溫)+말 | 따뜻한 정서, 고유어 느낌 | 뜻 설명이 한 번 필요 |

> 권장: **말씀** (1순위). 사용 예 — "말씀 키오스크", 시작 멘트 "말씀하세요, 제가 듣고 있어요."

**브랜드 보이스**: 공손한 존댓말 · 짧은 문장 · 외래어 0 · 재촉 없음. (예: "천천히 말씀하세요" O / "음성 인식 중입니다" X)

## 2. 데모 환경 (물리 전제)

- 화면: 10.1" 터치, **가로(landscape)**, 해상도 ⚠패널 스펙 확인(통상 1280×800 — 확인 후 본 문서 갱신)
- 시야거리: 탁상 거치, 약 50~70cm
- 입력: 터치 + 음성(Anker 스피커폰) / 출력: 화면 + 스피커
- 브라우저: Chromium 풀스크린 키오스크 모드 (커서·스크롤바 숨김)

## 3. 디자인 원칙 (연구 근거 기반)

> 출처: WCAG 2.2(대비 1.4.3, 텍스트 확대 1.4.4), Apple HIG·Material(터치 최소치), 고령층 UX 문헌(Toptal·Netguru·TPGi 등, 2025~2026). 핵심 합의 5가지를 우리 맥락에 적용.

1. **글자는 크고 또렷하게** — 문헌 공통 최소 16px는 *모바일 기준*이며, 우리는 50~70cm 거리의 10.1" 키오스크이므로 훨씬 큰 스케일을 쓴다(§4.2). 얇은 굵기 금지, 굵기 700 기본. "읽을 수 있다"가 아니라 "힘들이지 않고 읽힌다"가 목표.
2. **대비는 타협 안 함** — 노화로 대비 민감도·색 지각이 떨어지고 **파란색이 바래 보인다**. WCAG AA(4.5:1)는 최소선, AAA(7:1)를 목표로 하고 파랑 회피(§4.1). 회색 저대비 텍스트는 고령 UI에서 금물.
3. **아이콘엔 반드시 글자 라벨** — 아이콘 단독은 의미 전달 실패(예: 플로피=저장을 모름). 모든 주요 버튼은 그림+글자 동시. 색만으로 정보 구분 금지(색+글+아이콘 3중).
4. **즉각적이고 분명한 피드백** — 누르면 색 변화/애니메이션, 담기면 소리+음성. "방금 잘 됐다"를 항상 알려준다.
5. **재촉·복잡 제스처 없음** — 시간제한 금지(1분도 부족할 수 있음), 탭/단순 터치만(두 손가락·드래그·길게누르기 금지). 주요 기능은 항상 보이게(햄버거 메뉴식 숨김 금지) — 우리는 [처음으로]·[다시 듣기]·[글자 크게]를 전 화면 상단 고정.

**추가 적용**: ⑥ 시스템 글자 크기 존중 개념을 키오스크에선 [글자 크게] 토글로 제공(사용자가 가독성 제어). ⑦ 우리만의 강화 — 외래어 회피·우리말 우선(고령 외래어 이해 6.9%), 음성이라는 추가 입력 차원으로 문자 입력 부담 자체를 제거.

## 4. 디자인 토큰

### 4.1 컬러 (파랑 회피 — 고령 시인성)
| 토큰 | 값 | 용도 | 대비(목표) |
|---|---|---|---|
| bg | #FFF8EE | 바탕(따뜻한 크림) | — |
| ink | #221810 | 본문/제목 | bg 대비 ≥12:1 |
| sub | #6B5A48 | 보조 텍스트 | bg 대비 ≥5:1 |
| primary | #B6451F | 주 행동(말로 주문) | 흰 글자, 대형 텍스트 기준 ≥4.5:1 ⚠WebAIM 실측 |
| confirm | #2E5A34 | 확정(네, 맞아요) | 흰 글자 ≥6:1 ⚠ |
| card | #FFFFFF | 카드 | line #E6D9C8 |
| alert-bg | #FCEBE4 | 되묻기/안내 강조 | ink 텍스트 |

규칙: 흑백 변환에서도 구분 가능해야 함(인쇄 테스트 1회). 미달 색은 어둡게 조정.

### 4.2 타이포그래피
- 서체: **KoddiUD 온고딕** (한국장애인개발원 UD 서체, 무료·웹폰트) — 사용 사실을 발표에 명기
- 스케일(가로 1280 기준): 제목 56 / 큰버튼 36 / 본문·카드명 32 / 보조 24 / 최소 20 (이하 금지)
- 행간 1.4~1.5, 자간 0, **굵기 700 기본** (저시력 가독)
- 글자 크게 토글: ×1.25 전역 배율

### 4.3 레이아웃·간격
- 8pt 그리드. 좌우 여백 64. 콘텐츠 최대 폭 1152(중앙)
- 가로 화면 분할: 좌 60% 정보 / 우 40% 행동 버튼 컬럼 (확인·메뉴 화면), 단일 결정 화면은 중앙 1열
- 버튼: 높이 ≥88(주행동 ≥110), 간격 ≥16, 라운드 20
- 터치 영역 최소 72×72 — 고령층 권장치. 문헌은 48×48dp(Apple/Google 44보다 상향)를 권하고 "고령 대상은 권장치를 *최소선*으로 보라"고 명시하므로 우리는 그 이상을 적용(떨림·관절염 대응). 인접 타깃 간 충분한 여백으로 오터치 방지

## 6. 화면별 사양 (9) — 각 화면: 목적 / 핵심 요소 / TTS 멘트

1. **대기**: 끌어들이기 / 브랜드 로고+“화면을 누르거나 말씀하세요” / (PIR 감지 시) "어서 오세요. 무엇을 도와드릴까요?"
2. **시작**: 경로 선택 / 인사 헤드라인+예시 발화 1줄+[🎤 말로 주문][👆 그림 보고 누르기] / "말씀하시거나, 그림을 눌러 주세요."
3. **듣기**: 발화 수집 / 듣는 중 오브+실시간 자막+[다시 말하기]+[👆 그림 보고 누르기] / "천천히 말씀하세요."
4. **해석 중**: 대기 안정감 / 오브 색 전환+“알아듣는 중이에요” / (무음, 1.5s 내 전환 목표)
5. **되묻기**: 모호 해소 / 질문 1문장+후보 버튼 2~4 / 질문 그대로 TTS
6. **추천**: 망설임 해소 / "이건 어떠세요?" 헤드라인+추천 카드 1~3(사진+쉬운 이름+추천 이유 한 줄)+[이걸로 할게요][다른 거 보기] / "○○은 어떠세요? 달지 않고 부드러워요." (추천 이유를 음성으로)
7. **주문 확인**: 합의 / 시그니처 확인 카드+합계+[✓ 네][🎤 다시 말하기][👆 직접 고르기] / "○○ 하나, ○○ 하나 — 맞으세요?"
8. **메뉴**: 터치 완주 경로 / 카테고리 2탭+카드 그리드 3×2+하단 고정 [이대로 주문] / 담을 때마다 "○○ 담았어요."
9. **완료·호출**: 마무리 / 큰 체크+내역+수량±+[더 주문][계산 끝내기(직원 호출)] / "주문이 끝났어요. 직원을 불러 드릴게요."

전 화면 공통 상단: [⌂ 처음으로] [🔊 다시 듣기] [가 글자 크게]

## 7. 모션 & 사운드

- 전환 200~300ms ease-out, 슬라이드 최소화(페이드+살짝 위로)
- 펄스(듣기)·체크 드로잉(완료) 외 장식 모션 금지. `prefers-reduced-motion` 존중
- 사운드: 담기=짧은 1음(부드러운 목금), 완료/호출=상행 2음. 날카로운 비프 금지. 음량은 KS X 9211 조절 요건 따름
- TTS 보이스: 클라우드 한국어, **차분하고 또렷한 성인 보이스, 말속도 0.85~0.9** ⚠후보 2~3개 어르신 청취 비교로 선정(실증 전 미니 테스트)

## 8. 콘텐츠 (사진·카피)

- 메뉴 사진: **직접 촬영 권장**(저작권 안전) — 1:1, 밝은 단색 배경, 실물 중심, 최소 800×800. 톤 일관(자연광/백색)
- 금지: 스톡 워터마크, 과한 보정, 이모지 아이콘(제품감 저하)
- 카피 원칙: 한 문장 12자 내외 목표, 존댓말, 외래어 0 (메뉴 원어 병기는 예외)

## 9. 하드웨어 외관 (함체)

- 전면: 화면 베젤 + 하단 패널에 **로고(제품명)+“말씀하세요” 인쇄**, 마이크/스피커 타공
- 색: 크림/웜그레이 단색 + primary 포인트 1곳 (과한 그래픽 금지)
- 각도: 화면 상향 15~20° 탁상 스탠드(앉은 어르신 시선) ⚠실측
- 케이블 은닉, 뒷면 전원·포트 라벨. 운반 손잡이(경로당 이동)
- 예산: 잔여 ≈108,215원 내 (포맥스 재단+UV 인쇄 견적 ⚠)

## 10. 제품감 체크리스트 (시연 전 전부 ✓)

☐ 모든 화면 KoddiUD 적용·토큰 외 색 0건 ☐ 이모지 0건, 실사 100% ☐ 대비 검사 통과 캡처 보관 ☐ TTS 어르신 청취 테스트 통과 ☐ 함체 로고 인쇄 ☐ 부팅→풀스크린 무조작 진입 ☐ 30분 무중단 리허설 ☐ 사장님 기능(메뉴 CSV→시니어 모드) 시연 가능

## 11. Claude Design 투입 가이드

세션 시작 프롬프트(예):
> "10.1인치 가로(1280×800) 고령층용 음성 주문 키오스크 화면 8종을 디자인해줘. 첨부한 제품화 브리프의 토큰(컬러·KoddiUD 온고딕·사이즈)과 화면별 사양을 그대로 따라. 화면 1번(대기)부터 차례로, 한 화면씩 확정하고 넘어가자. 이모지 금지, 실사 사진 자리는 회색 플레이스홀더+라벨로."

작업 순서: 토큰 시트 → 컴포넌트 시트 → 화면 6(확인, 시그니처)부터 → 나머지 7종. 산출물: 화면 8 + 컴포넌트 1 + 함체 전면 1.
