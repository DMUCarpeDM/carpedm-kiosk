# 차별화 전략 & 리서치 의제
### CarpeDM — 고령층 AI 음성 가이드 키오스크 (대상 전략 문서)

> 목적: "배리어프리 키오스크가 이미 많은데 우리는 뭐가 다른가"에 답하고, 대상으로 가기 위해 무엇을 더 조사·실증할지 정리한다. 핵심 주장에 신뢰도(높음/중간/낮음)와 출처 표시.

---

## 1. 시장 분석 — 기존 배리어프리 키오스크는 무엇을 하나

| 사업자/사례 | 주요 기능 | 음성의 역할 |
|---|---|---|
| 코레일유통 (철도역) | 음성 출력, 안면 인식, 수어 영상, 점자, 고대비, 낮은 화면, 쉬운 용어·글자 크기 | 화면 읽어주기 + 시각장애인 키패드 주문 |
| 하나시스 (한국장애인개발원 보급) | 음성안내 키패드 주문, 고대비, 낮은 화면, 화면 확대; 접근성 검증 통과 | 화면 읽기 + 키패드 네비 |
| NICE인프라 | 낮은 화면·저시력·**음성주문 모드**; 국내 최초 접근성 검증(2023.6) | 음성주문 모드(정형 명령 추정) |
| NIA / 국립중앙박물관 | STT·TTS·수어 인식 + **의도 추론** 학습데이터셋 | 전시 안내(정부 대형사업) |
| 비버웍스 | 저가형, 핸드폰 미러링/QR오더 | (음성 비중 낮음) |

**(신뢰도: 높음 — 언론·기관 자료, 2025~2026)**

### 핵심 관찰
- 기존 음성은 대부분 ① **TTS로 화면 읽어주기(출력)**, ② **시각장애인용 키패드 네비게이션**, ③ **정형 음성 명령**이다.
- 즉 사용자가 여전히 **메뉴 구조와 정확한 명칭**을 알아야 한다.
- 주 타깃은 **장애인(특히 시각장애인) 법적 접근성 준수**. 고령층의 **인지 부담·표현 방식** 문제는 거의 다뤄지지 않는다.
- "접근성 인증 통과"가 상용 제품의 목표 → **정형화·표준화**에 최적화돼 있고, 고령층의 비정형 자연어는 사각지대.

## 2. 우리의 차별점 (3축 + 안전망)

> **포지셔닝 한 문장**: 기존 배리어프리 키오스크가 "장애인 접근성 준수 + 화면 읽어주기"에 머무는 반면, 우리는 **고령층 특화 + LLM 기반 '의미 이해형' 대화 주문** — 어르신이 메뉴명을 몰라도 *자기 말("검은 탄산 물", "부드러운 거")* 로 주문이 완성되는 키오스크.

| 축 | 내용 | 기존과의 차이 |
|---|---|---|
| ① 의미 기반 음성 주문 | LLM이 은유·간접·기능 표현을 의도로 해석 ("검은 탄산 물"→콜라) | 기존은 정형 명령/정확한 메뉴명 필요 |
| ② 고령층 표현 사전(데이터셋) | 구로구 현장에서 실제 표현 수집 → "표현→메뉴" 매핑을 프롬프트/검색에 반영 | 기존은 이런 자산 없음. **리빙랩 산출물·방어 자산** |
| ③ 대화형 보조 | 못 알아들으면 되묻고, 추천하고("부드러운 거"→카스텔라 추천), 어르신 페이스로 | 기존은 단계별 메뉴 네비게이션 |
| 안전망 | 음성 실패 시 큰버튼으로 완주 + 재확인 루프 | 법적 접근성(KS X 9211)도 동시 충족 |

**왜 방어 가능한가**: ②의 "고령층 표현 사전"은 현장 데이터로만 만들 수 있어 단순 복제가 어렵다. 대기업 상용 제품은 인증·표준화에 묶여 있어 비정형 자연어를 적극 수용하기 어렵다.

## 3. 핵심 기술 자산 — AI Hub 공개 데이터

| 데이터 | 설명 | 활용 |
|---|---|---|
| 노인 명령어 기반 한국어 음성 DB (AI Hub, dataSetSn=94) | 노인 음성 명령어 → 텍스트, **AI 키오스크 활용 명시** | STT 검증, 명령어 패턴 분석 |
| 자연어 자유대화 노인 음성 (AI Hub, dataSetSn=107) | 60세 이상 자유대화 음성+전사. **"노인은 사투리·억양 등 발화 특성이 달라 별도 음성인식 데이터 구축 필요"** 명시 | 발화 특성 근거, STT 정확도 평가, 표현 수집 보강 |

**(신뢰도: 높음 — AI Hub 공식)** ※ 다운로드는 승인 절차 필요. 라이선스·이용 범위는 사용 전 확인.

> 이 데이터의 존재 자체가 두 가지를 입증한다: (1) **고령층 발화는 일반과 다르다**(우리 문제 정의의 근거), (2) **STT만으로는 부족하다**(LLM 의미 해석이 필요한 이유).

## 4. 리서치 의제

조사를 셋으로 나눈다. **B(현장 1차)는 리빙랩의 핵심이라 팀이 직접 해야 하고, 외주·검색으로 대체 불가.**

|---|---|---|
| 페인포인트 통계 | 노인실태조사·서울디지털재단·소비자원 (기획서 1장) | ✅ 확보 |
| 시장/경쟁 | 기존 배리어프리 키오스크 기능 (본 문서 1장) | ✅ 확보 |
| 고령자 UI 가이드 | 6장 체크리스트 | ✅ 확보 |
| 공개 데이터셋 | AI Hub 노인 음성 (3장) | ✅ 확보 |
| 접근성 표준 | KS X 9211, 과기부 고시 | ✅ 확보 |
| 선행 연구 | 노인 키오스크·VUI 논문 추가 정리 | ☐ 팀 보강 |

### B. 현장 1차 조사 (구로구·금천구) — **최우선**
| 항목 | 방법 | 산출물 |
|---|---|---|
| 단계별 페인포인트 | 실제 매장에서 어르신 주문 관찰(어디서 막히는지) | 페인포인트 맵 |
| **고령층 표현 수집** | 어르신 인터뷰: "이걸 뭐라고 부르세요?" / 실제 주문 발화 녹음 | **표현 사전(핵심)** |
| 발화 특성 | 말 속도·사투리·말끝 흐림·되묻기 패턴 | STT 설계 입력 |
| 수용성 | 음성 주문에 대한 거부감/기대 | UX 방향 |

### C. 기술 검증 (PoC 실험)
| 항목 | 측정 | 목표 |
|---|---|---|
| STT 정확도 | 어르신 발화/소음 환경 인식률 | 실패 패턴 도출 |
| LLM 매핑 정확도 | 표현→메뉴 매핑 정답률 | 표현 사전 튜닝 |
| 대화 흐름 | 되묻기·복구 성공률 | 시나리오 개선 |

## 5. 고령층 표현 사전 — 시작 틀 (가설, 현장 검증 필요)

> ⚠ 아래는 **가설 예시**다. 실제 표현은 구로구 어르신 인터뷰로 수집·검증해야 하며(신뢰도: 낮음 — 미검증), 지역·세대마다 다를 수 있다.

| 어르신 표현(예시) | 의도(메뉴) |
|---|---|
| 검은 탄산 물 / 까만 사이다 | 콜라 |
| 뜨신 거 / 뜨거운 거 | 따뜻한 음료 |
| 시원한 거 / 찬 거 | 아이스 음료 |
| 부드러운 거 / 말랑한 거 | 카스텔라·푸딩·케이크류 |
| 안 매운 거 / 순한 거 | (옵션) 덜 맵게 |
| 달달한 거 / 단 거 | 단 음료·디저트 |
| 쓴 거 / 진한 커피 | 아메리카노 |
| 우유 든 커피 / 부드러운 커피 | 라떼 |

**수집 방법(인터뷰 질문 설계 예)**: ① "○○ 드시고 싶으면 키오스크에 뭐라고 하실 거예요?" ② 메뉴 사진 보여주고 "이건 뭐라고 부르세요?" ③ 실제 주문 상황 재현 녹음(동의 후).

## 6. 고령자 UI 설계 체크리스트 (근거 포함)

| 원칙 | 기준/근거 |
|---|---|
| 큰 터치 영역 | 버튼 최소 48×48px, 간격 12px 이상 (W3C 접근성 기준) |
| 고대비 + 파란 음영 주의 | 고령자에게 파란색은 희미하게 보여 대비 저하 → 대비 검사(WebAIM 등) |
| 단계적 공개 / one thing per page | 처리 속도가 느려 한 화면에 한 결정만; 중요 항목은 화면 중앙 |
| 시간 여유 | 타임아웃 없음, 재촉 없음 (페인포인트 '뒷사람 눈치' 대응) |
| 실수 방지·복구 | 되돌리기 쉽게; "실수할까봐 불안해 아무것도 안 하는" 이탈 방지 |
| 최소 입력 | 과도한 핑거 태핑 회피 → 음성으로 대체 |
| 3중 표현 | 글씨 + 음성 + 그림(픽토그램) 병행; 색만으로 구분 금지 (KS X 9211) |

**(신뢰도: 높음 — UX 가이드 다수 + 국가표준)**

## 7. 추가 기능 후보 (대상 노림)

| 기능 | 효과 | 우선순위 |
|---|---|---|
| **메뉴 추천** | "부드러운 거" → 후보 제시. 메뉴를 몰라도 주문 | 높음 (차별축 ③) |
| **오류 복구 대화** | 인식 실패 시 되묻기 + 후보 버튼 | 높음 (신뢰성) |
| **음성 속도·크기·다시듣기** | KS X 9211 충족 + 고령 배려 | 높음 |
| **결제 보조** | 결제는 어려움 → "직원 호출" 또는 간편 안내 | 중간 (실증 환경 고려) |
| 단골/반복 주문 기억 | 자주 시키는 메뉴 빠른 재주문 | 낮음 (선택) |
| 방언 대응 강화 | 지역 사투리 표현 추가 | 중간 (확장) |

> **과범위 경고**: 학생 PoC 단계에서 **실제 결제·POS 연동은 제외**하고 '주문 완료까지'를 데모 범위로. 결제는 직원 호출/모의 처리로 충분.

## 8. 왜 대상감인가 (심사 매핑)

| 심사 관점 | 우리의 강점 |
|---|---|
| 신규성·독창성 | 의미 기반 VUI + 고령층 표현 데이터셋 — 기존 상용/공공이 안 하는 영역 |
| 리빙랩 적합성 | 현장에서 표현 수집 → 시스템 반영 → 재실증의 **완결된 리빙랩 루프** |
| 사회적 임팩트 | 고령 밀집지(구로구 22.2%) 디지털 포용 + 정책 제언 |
| 학술·확장 자산 | 표현 데이터셋 자산화, 공공 인프라·다문화 확장 |
| 실현 가능성 | AI Hub 데이터 + LLM API로 학생팀이 구현 가능 |
