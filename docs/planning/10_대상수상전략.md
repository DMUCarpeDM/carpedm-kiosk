# 10. 대상 수상 전략 — 실증 직후 격차 진단과 마지막 주 실행 계획

> 작성: 2026-07-11 (실증 종료 다음 날). 근거: 저장소 전수 감사 + 실증 로그 분석 +
> 기획 문서(01·02·05·08·09) 재검토 + 리빙랩/캡스톤 심사 기준 공개 자료 조사.
> 원칙: 정직성 원칙 유지 — 실측과 제안치를 구분하고, 없는 수치를 만들지 않는다.

## A. 지금 어디에 있나 (한 줄 진단)

**기술 완성도와 스토리는 대상급인데, "실증을 숫자로 증명하는 마지막 고리"가 아직 저장소에 없다.**
다음 주(7/13~17)의 승부는 코딩이 아니라 **데이터 회수 → 재측정 → 발표 자산화**다.

### 이미 갖춘 것 (대상급 강점)
- 해석 정확도 96.3% / 멀티턴 95.7% (테스트셋 246건, 목표 90%/85% 초과) — `reports/accuracy.md`
- 규칙 폴백 98.8% + 3중 폴백 계층(Claude→규칙→터치) — 오프라인에서도 주문 완결
- 환각 차단 설계(LLM은 메뉴 id만 반환), PII 없는 로깅을 테스트로 검증 — 심사 Q&A 방어 자산
- 태블릿(iPad·갤럭시탭) 실기기 구동 + 라이브 데모 가능한 드문 주제
- 경쟁 분석 기반 포지셔닝("의미 이해형" vs 기존 "화면 읽어주기") — `02_차별화전략.md`

### 없는 것 (대상을 막는 격차)
1. **실증 데이터가 이 저장소에 없다.** `data/logs/utterances.jsonl`의 실증 기간 항목은 18건뿐이고
   그중 11건이 동일한 리허설 문장, 7/6(D-day) 로그는 0건. **이 로그를 "실증 참여자 데이터"로
   발표하면 안 된다.** 실제 데이터는 파이/태블릿 서버의 로그 + 종이 기록지·SUS·표현수집표에 있을 것.
2. **표현 사전이 아직 '가설판'이다.** `data/expressions.yaml` 헤더에 명시된 대로 현장 표현으로
   교체·재측정해야 하는데, 이 "교체 전/후 정확도 비교"가 실증키트가 정의한 **보고서 핵심 그래프**다.
3. **로컬 18커밋이 원격에 없다.** `feature/tablet-support`가 origin/main보다 18커밋 앞서 있고
   태블릿·카메라 인체감지·접근성 보강 등 최근 일주일 작업 전체가 이 노트북에만 존재. 발표 주간에
   기기 문제가 나면 시연 빌드를 통째로 잃는다.

## B. 심사에서 대상을 가르는 것 (공개 자료 조사 결과)

동양미래대 자체 심사표는 비공개(RISE사업단에 확인 권장). 공개된 유사 루브릭의 공통 구조:
**문제정의·창의성 20~35% / 실증·완성도 20~30% / 지역사회 기여 20% / 발표·질의응답 20~25%**
(경희대 LINC 3.0·숙명여대·KSCE 캡스톤 공고 기준. 경희대는 "지역사회 연계"가 명시적 가산점.)

리빙랩 성과공유회 대상 수상작의 공통점 (제주관광대·경복대 2025~26 사례, STEPI 리빙랩 연구):

| 변별 요소 | 대상 수상작이 한 것 | 우리의 대응 |
|---|---|---|
| 수치화된 실증 | "고령층 50명 실증, 전 항목 목표치 초과"를 숫자로 | 목표치(현장 1차 발화 ≥75%, 복구 포함 완료율 ≥95%) 대비 실측치 표 |
| 주민 = 공동 설계자 | 피드백→설계 변경 사례 제시 | "어르신 표현 수집→사전 반영→정확도 상승" 전/후 그래프 + 개선 사례 2~3개 |
| 산출물 3종 | 작동 프로토타입 + 실증 리포트 + 정책 제안서 | 프로토타입 ✅ / 리포트(6장 갱신) / 정책 제언 절 보강 |
| 지속가능성 | 확산 로드맵, 후속 연계 | 구로구 내 확산·병설형 도입 시나리오, 표현 데이터셋 자산화 |
| 발표·시연 | 실물 시연 + 어르신 실사용 영상 | 라이브 데모(심사위원이 직접 음성 주문) + 백업 영상 |

리빙랩 심사에는 지자체·복지관 등 **비전공 심사위원**이 흔히 포함된다(한국해양대 사례).
기술 용어보다 "구로구 어르신의 줄이 어떻게 사라지는가"로 말할 것.
