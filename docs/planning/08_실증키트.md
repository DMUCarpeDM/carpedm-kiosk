# 실증 운영 키트 v1 — "언제든 찾아가면 바로 시작"
### CarpeDM 고령층 AI 음성 키오스크 · 방문형 실증용 휴대 문서

> 용도: 복지관·경로당·주민센터 등 어디든 방문해 즉시 실증을 진행하기 위한 인쇄용 키트.
> 구성: ① 출발 전 체크리스트 ② 진행 스크립트 ③ 과업 카드 ④ 기록지 ⑤ SUS ⑥ 동의서 ⑦ 표현 수집표 ⑧ 데이터 회수
> ⚠ 동의서(⑥)는 사용 전 지도교수(조진형 교수님) 검토를 받을 것. 법률 자문이 아닌 초안이다.

---

## 0. 목표·표본 설계
- 미니 파일럿: 2~3명 (스프린트2 중, 키트 리허설 + 표현 사전 1차 교체)
- 본실증: 8~12명 (7/6~7/10), 1인당 8~10분
- 산출 데이터: 과업 성공률·소요시간, 1차 매핑 성공/되묻기/복구(자동 로그+기록지), SUS, 표현 수집(사전 교체용)
- 현장 예절: 가능하면 전날 시설 책임자에게 전화 1통, 도착 시 학생증 제시 + 1분 소개

## ① 출발 전 체크리스트 (10분)
- 기기: 본체·전원 어댑터·**멀티탭 3m**·스피커폰(충전 확인)
- 네트워크: 휴대폰 핫스팟 데이터 잔량 확인 → 연결 테스트 / **폴백(rule) 동작 1회 확인** (와이파이 없어도 진행 가능해야 함)
- SW: 부팅→키오스크 모드 자동 진입, `/healthz` OK, 음량 70%, 테스트 주문 1건 후 `data/logs/utterances.jsonl` 줄 추가 확인 → **테스트 로그는 표시해두기**(분석 시 제외)
- 서류: 동의서 15부, 기록지 15부, SUS 척도 카드(큰 글씨 1~5), 볼펜 3자루, 답례품(예산 내 결정)
- 역할: 진행자 1(말 걸기), 기록자 1(기록지), 기술 1(장비·로그)

## ② 진행 스크립트 (진행자가 그대로 읽기)
**인사**: "안녕하세요, 동양미래대학교 학생들입니다. 어르신들이 쓰시기 쉬운 '말로 주문하는 기계'를 만들고 있어요. 한 번 써보시고 느낌을 말씀해 주시면 큰 도움이 됩니다. **잘하고 못하고는 전혀 없습니다 — 기계가 못 알아들으면 그건 기계 잘못이고, 저희가 고칠 숙제예요.**"
**동의**: 동의서를 천천히 읽어드리고 서명(또는 대리 기록) → "지금부터 말씀하시는 내용이 녹음됩니다" 고지 → 시작.
**진행 규칙**: 힌트 금지. 막히면 "기계에게 다시 한 번 말씀해 보세요" 1회만. 연속 2회 실패 시 화면의 **큰 그림 버튼(터치)** 안내 허용(이것이 '복구 경로' — 실패가 아니라 측정 항목). 키보드 입력은 없음.
**종료 질문(표현 수집)**: "감사합니다. 평소 이런 기계에서 뭐라고 말하고 싶으셨어요? 자주 시키시는 건 뭐라고 부르세요?"

## ③ 과업 카드
- **T1 자유 주문**: "지금 드시고 싶은 것을, 평소 말씀하시는 대로 이 기계에 말해서 주문해 보세요."
- **T2 지정 주문**: "**시원한 커피 한 잔과 단팥빵 하나**를 말로 주문해 보세요." (전원 동일 — 비교 기준)
- **T3 터치 주문(선택)**: 같은 주문을 화면 버튼만으로. (음성 vs 터치 시간 비교용)

## ⑤ SUS 만족도 (구두 진행 — 진행자가 읽고, 큰 글씨 척도 카드를 가리키게 함)
1=전혀 아니다 ~ 5=매우 그렇다
1. 이 기계를 자주 사용하고 싶다
2. 이 기계는 불필요하게 복잡하다
3. 이 기계는 사용하기 쉬웠다
4. 이 기계를 쓰려면 옆에서 도와줄 사람이 필요할 것 같다
5. 이 기계의 기능들이 잘 어우러져 있다
6. 이 기계는 일관성이 없는 부분이 많다
7. 대부분의 사람들이 사용법을 금방 배울 것 같다
8. 이 기계는 사용하기에 매우 번거롭다
9. 이 기계를 사용하는 데 자신감이 들었다
10. 쓰기 전에 배워야 할 것이 많았다

채점: 홀수 문항(점수−1) + 짝수 문항(5−점수), 합계 ×2.5 = 0~100점.
⚠ 응답 부담이 크면 1·3·4·9번만 진행하고 보고서에 "축약 시행(참고치)"로 명시 — 표준 SUS와 혼용 집계 금지.

## ⑥ 참여 동의서 (A4 1장 · 큰 글씨 인쇄 · ⚠지도교수 검토 후 사용)
**'말로 주문하는 키오스크' 사용성 평가 참여 동의서**

- 목적: 동양미래대학교 학생팀(CarpeDM)의 리빙랩 연구 — 어르신이 쓰기 쉬운 음성 주문 기계 개선
- 활동: 기계로 모의 주문 체험 약 5~10분 (실제 결제 없음)
- 수집 정보: 음성 녹음과 말씀하신 문장, 연령대, 키오스크 사용 경험. **이름·전화번호 등 개인 식별정보는 수집하지 않습니다.**
- 활용: 연구 보고서·발표 자료, '어르신 표현 사전' 제작. 모든 내용은 익명 처리됩니다.
- 보관·파기: 자료는 팀 내에만 보관하며 과제 종료 후 1년 이내 파기합니다.
- 자발성: 참여는 자유이며, 언제든 중단하실 수 있고 어떤 불이익도 없습니다.
- 사진·영상 촬영(선택): □ 동의함 □ 동의하지 않음 (동의 시에도 얼굴은 알아볼 수 없게 처리)
- 문의: 동양미래대학교 인공지능소프트웨어학과 (지도교수 조진형)

참여자 서명: ________ (날짜: ____)
※ 서명된 동의서 원본은 기록지·데이터와 **분리 보관**하며, 데이터에는 참여자 번호만 사용합니다.

## ⑦ 표현 수집표 (리빙랩 핵심 자산 — 사전 교체용)
| 들은 표현 (그대로) | 의도한 메뉴 | 상황 메모 |
|---|---|---|
| 예: "검은 탄산 물" | 콜라 | T1 중 |
|  |  |  |

귀가 후: `data/expressions.yaml`에 반영 → `python scripts/measure.py` 재실행 → **교체 전/후 정확도 비교** 기록 (보고서 핵심 그래프).

## ⑧ 당일 데이터 회수 (귀가 후 30분)
1. `utterances.jsonl` 백업(날짜 폴더) — 테스트 로그 제외 표시 확인
2. 기록지 스캔/촬영, 동의서 원본 분리 보관
3. 녹음 파일 익명 폴더 이동
4. 표현 수집표 → 사전 반영 → measure 재실행 → 결과 메모
