## 1. 우리가 만드는 것 (한 문장)

**어르신이 메뉴 이름을 정확히 몰라도, "검은 탄산 물 줘" 처럼 자기 말로 주문하면 알아듣는 키오스크.**

- 대상: 키오스크가 어려운 고령층 (구로구·금천구)
- 형태: 기존 키오스크 옆에 놓는 **음성 주문 보조 창구** (기계 하나로 주문 완결, 결제는 직원 호출)
- 핵심 차별점: 메뉴판의 어려운 외래어를 몰라도 **일상적인 말**로 주문됨

---

## 2. 어떻게 작동하나 (3단계)

```
🎤 어르신이 말함
   "시원한 거 달달한 커피 줘"
        │
        ▼
 ① STT  : 말 → 글자로 바꿈        ("시원한 달달한 커피")   ← 아직 작업 전
        │
        ▼
 ② 해석  : 글자 → 정확한 주문으로  (→ 아이스 카페라떼 1잔)   ← ✅ 완성·검증됨
        │
        ▼
 ③ TTS  : 글자 → 음성으로 안내    ("아이스 라떼 맞으세요?") ← 아직 작업 전
```

**중요**: 우리 프로젝트의 두뇌이자 가장 어려운 ② 해석 부분을 먼저 완성했습니다. ①③은 외부 음성 서비스를 "연결"하는 작업이라 상대적으로 정형화돼 있습니다.

1. **현장 데이터로 검증** — 다른 팀은 데모만, 우리는 실제 어르신 발화로 정확도를 숫자로 증명

4. **"팔아도 될" 완성도** — 브랜드·디자인까지 갖춘 제품 수준

## 6. 자주 나오는 질문 (팀원·심사 대비)

**Q. AI가 다 만든 거 아닌가요?**
A. AI는 도구일 뿐입니다. 우리가 한 건 ① 어떤 문제를 풀지 정의 ② 어르신 표현을 수집·정리 ③ 정확도를 측정하고 개선하는 체계 구축 ④ 현장 검증. 이건 AI소프트웨어학과의 전공 역량 그 자체입니다.

**Q. 기존 배리어프리 키오스크랑 뭐가 다른가요?**
A. 기존 건 글자를 읽어주거나 화면을 키우는 수준입니다. 우리는 어르신이 **메뉴명을 몰라도** 일상 언어로 주문하게 합니다. ("아메리카노" 대신 "쓴 커피"라고 해도 됨)

**Q. 결제는 어떻게 하나요?**
A. 이번 단계에선 주문까지만 하고 직원을 호출합니다 (실제 결제 연동은 장기 과제). 어르신의 주문 부담을 더는 게 핵심이라 이걸로 충분합니다.

**Q. 인터넷이 끊기면요?**
A. 인터넷 없이도 작동하는 백업 모드가 있습니다 (정확도는 조금 낮아지지만 주문은 가능).
