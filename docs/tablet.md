# 태블릿(iPad·갤럭시탭) 키오스크 실행 가이드

> Pi5 없이 아이패드·갤럭시탭을 키오스크 화면으로 쓰는 방법.
> 구조는 동일: 태블릿 = 화면+마이크+스피커(thin client), 서버(맥 또는 파이) = 백엔드+프론트 서빙.

```
[태블릿: Safari/Chrome 전체화면]
      │  같은 Wi-Fi, HTTPS (마이크 권한 필수 조건)
[서버(맥/파이): FastAPI + 빌드된 프론트, 단일 포트 8443]
      │
[클라우드: CLOVA STT · Claude · Google TTS]
```

**왜 HTTPS가 필요한가** — 브라우저는 마이크(getUserMedia)를 `localhost` 또는
HTTPS에서만 허용한다. 태블릿은 서버에 LAN IP로 접속하므로(localhost가 아님)
HTTPS가 없으면 음성 주문이 아예 동작하지 않는다.

## 1. 인증서 준비 (서버에서 1회)

```bash
brew install mkcert nss   # 맥. 파이는: sudo apt install mkcert (또는 openssl 폴백 사용)
bash scripts/make-tls-cert.sh
```

- `secrets/tls/kiosk.pem`, `kiosk-key.pem` 생성됨 (서버용)
- `secrets/tls/rootCA.pem` 생성됨 (태블릿에 설치할 루트 인증서)

**주의: Wi-Fi가 바뀌어 서버 IP가 달라지면 스크립트를 다시 실행**해서 새 IP가 포함된
인증서를 만들어야 한다. 실증 장소 도착 후 한 번 더 돌리는 것을 권장.

### 태블릿에 루트 인증서 설치 (mkcert 사용 시, 권장)

**iPad**
1. `rootCA.pem`을 AirDrop·이메일로 iPad에 전송 → 열기 → "프로파일 다운로드됨"
2. 설정 > 일반 > VPN 및 기기 관리 > 다운로드된 프로파일 > 설치
3. 설정 > 일반 > 정보 > 인증서 신뢰 설정 > **mkcert 루트를 "전체 신뢰" 켜기** ← 잊기 쉬움
4. 실증 끝나면 같은 경로에서 프로파일 삭제

**갤럭시탭**
1. `rootCA.pem`을 태블릿으로 전송 (USB·이메일)
2. 설정 > 보안 및 개인정보 보호 > 기타 보안 설정 > 인증서 설치 > CA 인증서 > 설치

### mkcert 없이 (openssl 자가서명 폴백)

인증서 설치 생략 가능. 대신 태블릿 첫 접속 때 보안 경고가 뜨면
**고급 > 계속 진행**으로 통과한다. 통과 후에는 마이크·오디오 모두 정상 동작한다.
(iPad Safari는 경고 통과 후에도 마이크를 막는 경우가 드물게 있어 mkcert 방식 권장.)

## 2. 서버 실행

### 실증(운영) 모드 — 단일 포트, 권장

```bash
bash scripts/serve-tablet.sh          # 빌드 + https://<서버IP>:8443
```

### 개발 모드 — HMR 필요할 때

```bash
# 터미널 1: 백엔드
.venv/bin/uvicorn backend.app:app --port 8001

# 터미널 2: 프론트 (LAN 노출 + HTTPS)
cd frontend
KIOSK_BACKEND=http://127.0.0.1:8001 \
KIOSK_TLS_CERT=../secrets/tls/kiosk.pem \
KIOSK_TLS_KEY=../secrets/tls/kiosk-key.pem \
npx vite --port 5174
# → 태블릿에서 https://<서버IP>:5174
```

이 맥에서는 5173·8000 포트를 다른 프로젝트(EXPO)가 쓰므로 위처럼 8001·5174 사용.

## 3. 태블릿 설정

1. 서버와 **같은 Wi-Fi** 접속 (공용 Wi-Fi는 기기 간 통신(AP isolation)이 막힌 경우가
   있음 — 안 되면 폰 핫스팟에 서버·태블릿을 함께 물리는 게 가장 확실)
2. Safari(iPad)/Chrome(갤럭시탭)에서 `https://<서버IP>:8443` 접속
3. 마이크 권한 요청 → **허용** (iPad는 첫 녹음 버튼 터치 때 뜸)
4. **홈 화면에 추가** → 홈 화면 아이콘으로 실행하면 주소창 없는 전체화면 실행
   - iPad: 공유 버튼 > 홈 화면에 추가
   - 갤럭시탭: Chrome 메뉴 ⋮ > 홈 화면에 추가
5. 실행 후 **화면을 한 번 터치**해야 안내 음성이 나온다 (브라우저 자동재생 정책 —
   대기 화면 "화면을 터치하세요"가 그 역할을 함)

### 키오스크 모드 고정 (실증 현장 권장)

관람객이 앱을 빠져나가지 못하게 잠근다.

- **iPad — 사용법 유도(Guided Access)**: 설정 > 손쉬운 사용 > 사용법 유도 켬 →
  키오스크 화면 실행 → 상단 버튼(또는 홈 버튼) 3번 클릭 → 시작.
  해제도 3번 클릭 + 암호.
- **갤럭시탭 — 앱 고정**: 설정 > 보안 > 기타 보안 설정 > 앱 고정 켬 →
  최근 앱 버튼 → 앱 아이콘 터치 > 이 앱 고정.

### 화면 꺼짐 방지

- iPad: 설정 > 디스플레이 및 밝기 > 자동 잠금 > **안 함**
- 갤럭시탭: 설정 > 디스플레이 > 화면 자동 꺼짐 시간 최대 (또는 개발자 옵션 > 충전 중 화면 켜짐 유지)

## 4. 동작 확인 순서

1. `https://<서버IP>:8443/healthz` → `{"ok": true...}` 확인
2. 메인 화면 진입, 터치 주문 흐름 확인
3. 음성 주문: 버튼 누르고 말하기 → 인식·응답·TTS 재생 확인
4. 큰 글씨/낮은 화면 모드 토글 확인

## 5. 문제 해결

| 증상 | 원인·해결 |
|---|---|
| 마이크 권한 팝업이 아예 안 뜸 | HTTP로 접속했거나 인증서 미신뢰 → 주소가 `https://`인지, 자물쇠 아이콘 확인 |
| "연결이 비공개가 아닙니다" | 자가서명 폴백 → 고급>계속. mkcert면 루트 CA 설치+iPad '전체 신뢰' 확인 |
| 접속 자체가 안 됨 | 같은 Wi-Fi인지, 서버 IP 변경 여부(재발급), 맥 방화벽(설정>네트워크>방화벽에서 Python/node 허용) |
| 소리가 안 남 | iPad 측면 무음 스위치/볼륨, 그리고 화면을 한 번이라도 터치했는지 |
| 음성이 찌그러져 인식 실패 | iPad 오디오 세션 문제 → 페이지 새로고침 후 재시도 (재현되면 이슈로 기록) |
| 화면이 위아래로 튕김(바운스) | 홈 화면 추가 아이콘으로 실행하면 사라짐 (브라우저 탭에서는 일부 남을 수 있음) |

## 6. Pi5와의 관계

Pi5 배포(docs/raspberry-pi.md)는 그대로 유효하다. 이 문서는 클라이언트만 태블릿으로
바꾼 것 — 백엔드·프론트 코드는 동일하고, 서버는 맥이든 파이든 상관없다.
파이를 서버로 쓰고 태블릿을 화면으로 쓰는 조합도 같은 방법으로 동작한다.
