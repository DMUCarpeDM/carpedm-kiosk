# 공공 데이터 활용 계획 v1
### "공공 데이터로 보강하고, 현장 데이터로 증명한다"

> 위치: 코드베이스 `docs/`. 관련 도구: `scripts/mine_expressions.py`, `scripts/eval_stt_cer.py`, `scripts/augment_testset.py`
> 원칙: 공공·합성 데이터는 **보강**이다. 리빙랩의 핵심 증거(표현 사전 교체 전/후, 현장 지표)는 실증에서 나온다.

## 2. 세 가지 용도와 절차

**용도 A — STT 사전 평가 (실증 전에 '귀'를 검증)**
1. 데이터셋 107에서 음성 샘플 ~100개 + 정답 전사 추출
2. ① 클라우드 STT ② faster-whisper(small, 파이5)로 각각 인식 → (ref, hyp) CSV 2개
3. `python scripts/eval_stt_cer.py --csv reports/stt_cloud.csv` → 평균 CER 비교표
4. 산출: **"노인 음성 기준 STT 선택 근거"** 표 + 최악 케이스 패턴(되묻기 설계 반영)

**용도 B — 표현 후보 마이닝 (사전 v0.5)**
1. 라벨 JSON 폴더에 `python scripts/mine_expressions.py --input <폴더>` 실행
2. 후보 CSV를 엑셀로 검수(의도 메뉴 id 기입) → `data/expressions.yaml` 반영
3. `python scripts/measure.py` 로 **반영 전/후 정확도** 기록
4. 한계 정직: 자유대화 도메인이라 주문 표현 밀도는 낮을 수 있음 — 건지는 만큼만 쓰고, 본 사전 교체는 현장 수집으로

**용도 C — 합성 증강 (강건성 측정)**
1. `python scripts/augment_testset.py --n 2` → 어르신 화법 변형 발화 생성(군말·간접 표현, 희화화 금지)
2. `measure.py --testset data/testset_augmented.jsonl` 로 강건성 측정
3. 보고서에는 **원본 100건 결과와 합성 결과를 반드시 분리 표기** (source 필드로 구분)

## 3. 데이터 윤리·라이선스 체크
- AI Hub 데이터: 승인 목적 내 사용, **원데이터·파생 음성 재배포 금지**, 보고서에는 출처와 데이터셋 번호 명기
- 합성 데이터: 생성 모델·프롬프트를 보고서에 공개(재현성), 캐리커처식 사투리 금지 지시 포함됨
- 현장 데이터: 실증 키트의 동의서 절차 준수, 익명 처리

## 4. 보고서 서술 예시 (방법론 챕터)
> "공공 노인 음성 데이터(AI Hub, dataSetSn=107)로 STT 엔진을 사전 평가해 선택 근거를 마련하고, 전사 말뭉치 마이닝과 LLM 합성 증강으로 표현 사전과 테스트셋을 보강했다. 최종 사전은 현장 실증에서 수집한 실제 표현으로 교체했으며, 교체 전/후 매핑 정확도 변화를 측정해 리빙랩 효과를 정량화했다."
