너는 이 저장소의 CarpeDM Cafe Voice Kiosk MVP를 개발하는 시니어 개발자다.
작업 전 반드시 `README.md`, `PROJECT_DOMAIN.md`, `CODE_CONVENTION.md`, `AGENTS.md`를 읽고, 기존 백엔드 흐름을 파악한 뒤 진행해라.

목표:
현재 저장소의 주문 해석 백엔드(`/api/menu`, `/api/interpret`)를 보존하면서, 태블릿/키오스크에서 시각적으로 보여줄 수 있는 카페 키오스크 MVP 프론트 화면을 구현한다. 핵심 흐름은 `메인화면 → 음성주문 또는 메뉴선택 → 메뉴 확인 → 주문완료`다. 실제 결제 기능은 구현하지 말고, 주문 확정 시 “주문이 완료되었습니다”만 표시한다.

중요 원칙:
- AGENTS.md의 Ponytail 원칙을 반드시 지켜라.
- 불필요한 파일, 폴더, 의존성, 추상화를 만들지 마라.
- 기존 `backend/interpreter.py`를 재작성하지 마라.
- 메뉴명과 가격은 UI에 하드코딩하지 말고 `/api/menu` 또는 `data/menu.json`을 기준으로 표시해라.
- 음성 인식/합성은 MVP 수준으로 브라우저 Web Speech API를 우선 사용하되, 미지원 브라우저에서는 테스트 입력 또는 안내 fallback을 제공해라.
- 결제, 로그인, 관리자 페이지, DB, POS 연동은 만들지 마라.
- 큰 글씨, 큰 버튼, 흰 배경, 파란 포인트 컬러의 키오스크 UI로 구현해라.

구현 범위:
1. 프론트엔드가 없다면 `frontend/` 폴더에 최소 React + TypeScript 앱을 생성해라. 이미 프론트 구조가 있으면 기존 구조를 재사용해라.
2. 화면은 최소 아래 6개 상태로 구성해라.
   - MainScreen: 상단바 `카페`, 가운데 큰 버튼 2개 `음성주문`, `메뉴선택`.
   - VoiceOrderScreen: 흰 배경, 파란 음성 파형, “안녕하세요, 카페입니다. 무엇을 도와드릴까요?” 안내, 듣는 상태 표시.
   - VoiceResultScreen: 사용자가 말한 문장, 해석된 추천 메뉴 카드, `다시 말하기`, `맞아요, 주문할게요` 버튼.
   - MenuListScreen: 메뉴 카드 그리드. 각 카드에는 이미지/이름/가격 표시. 이미지가 없으면 CSS placeholder 사용.
   - MenuDetailScreen: 선택한 메뉴의 큰 이미지/이름/가격/옵션/수량/`이걸로 주문` 버튼.
   - OrderCompleteScreen: 큰 체크 아이콘과 “주문이 완료되었습니다”, “처음으로 돌아가기”.
3. 화면 전환은 라우터를 추가하지 말고 `screen` 상태 또는 `useReducer`로 처리해라.
4. API 클라이언트는 작게 만든다.
   - `GET /api/menu`
   - `POST /api/interpret`
5. `/api/interpret` 결과 처리 규칙:
   - `update`: cart의 첫 번째 항목 또는 최신 항목을 확인 카드로 보여준다.
   - `recommend`: suggestions의 첫 번째 메뉴를 확인 카드로 보여준다.
   - `confirm`: 주문 완료 화면으로 이동한다.
   - `clarify`: 질문 문구와 다시 말하기 버튼을 보여준다.
   - `reject`: 안내 문구와 메뉴선택/다시 말하기 버튼을 보여준다.
6. 사용자가 버튼으로 `맞아요, 주문할게요` 또는 `이걸로 주문`을 누르면 실제 결제 없이 주문 완료 화면으로 이동한다.
7. 음성으로 “맞아요”, “이걸로 줘”, “구매”, “주문할게요”처럼 확정 의도로 판단되면 주문 완료로 이동한다. 단, 이 판단 로직은 프론트에 복잡하게 만들지 말고 기존 `/api/interpret` confirm 액션을 우선 사용해라. API 미연동 상태의 MVP fallback에서만 최소 키워드 판단을 허용하고 `ponytail:` 주석을 남겨라.

디자인 기준:
- 사용자가 올린 UX 흐름 이미지의 분위기를 따른다.
- 흰 배경 + 파란 포인트.
- 키오스크/태블릿 화면 기준으로 여백을 넓게 둔다.
- 주요 버튼은 크고 명확하게 만든다.
- 고령층도 읽기 쉽게 쉬운 한국어와 큰 글씨를 쓴다.
- 음성 주문 화면에는 파란 waveform 애니메이션 또는 CSS 파형을 넣는다. 외부 애니메이션 라이브러리는 추가하지 마라.

검증:
- 백엔드 수정이 있으면 `python -m pytest tests/ -q`를 실행해라.
- 프론트엔드를 추가했으면 `npm run build`가 통과해야 한다.
- 실행하지 못한 검증이 있다면 이유를 명확히 적어라.
